01

Load the extension

Unpacked, from the built output. Build first with npm run build.

Chrome / Edge / Brave / Arc

  1. Open chrome://extensions
  2. Toggle Developer mode (top-right)
  3. Load unpacked → select dist/chrome
  4. Pin Lockd to the toolbar

Firefox

  1. Open about:debugging#/runtime/this-firefox
  2. Load Temporary Add-on…
  3. Select dist/firefox/manifest.json
  4. Reload it after a Firefox restart (temporary add-ons clear)
↻

After changing code: run npm run build, then click reload on the extension card (Chrome) or Reload (Firefox).

02

The 60-second smoke test

Confirms the whole loop is alive before you dig in.

  1. Click the toolbar icon → popup shows idle, --:--, "No active session".
  2. Click Open dashboard →. Dashboard opens in a tab (dark theme).
  3. Top-right: click ◑ Light then ◐ Dark — the whole UI reskins and the choice sticks.
  4. Click New session → pick Test duration + Test rhythm → type an intent → Lock in →.
  5. You land in Work with a live countdown; it auto-flips work → break → work and finishes on the Session complete roll-up.
ExpectIf that works, the core engine, UI routing, theming, and the session loop are all functioning.

03

Feature-by-feature tests

Each block: what it is, how to drive it, and what to expect.

A · Sessions & the work/break loop

Timed work blocks (locked) alternating with breaks (unlocked), auto-resuming until the total duration is used up.

  1. Dashboard → New session → Test duration (~30s total) + Test rhythm (~6s work / ~4s break) → Lock in.
ExpectTimer counts down; pill reads "Focus locked" in work and "Everything unlocked" in break; the view auto-switches; ends on the roll-up.
!

Keep the dashboard or popup open during Test runs. Sub-minute transitions are nudged by the open UI; with everything closed the browser can delay very short alarms.

B · Calm vs Command work surface

Two work UIs. Calm = giant 160px timer, nothing else. Command = smaller timer + live stat tiles + a 28-day focus wall.

  • Pick 🛰️ Command in Setup, or use the Calm / Command toggle at the top of the Work view mid-session.
ExpectCommand reveals four stat tiles + the focus-wall grid; Calm hides them. Instant switch, persists.

C · Count direction toggle

Show time left (countdown) or focused (count-up).

  • In Work view, click Show focused / Show time left.
ExpectThe big timer flips between remaining and elapsed for the current block.

D · Pause / resume / skip / extend

  • Work → Pause: timer freezes, pill shows "Paused", button becomes Resume.
  • Break → +5 min: adds time. Skip break → jumps to the next work block.
ExpectPause is a soft control (no friction). Skipping a work block earns no XP/credit (anti-cheat).

E · Element-level blocking the differentiator

During work, curated CSS hides distraction elements while keeping the site usable. Defaults on: YouTube home feed, recommendations, Shorts, end-screen; Instagram Reels/Explore; X "For You"/trending/explore; Reddit feed/popular nav. (YouTube comments and Instagram suggested default off.)

  1. Start any session (popup Quick start works).
  2. During a work block, open youtube.com → homepage feed gone; open a video → recommendations sidebar & Shorts gone.
  3. In dashboard Distraction presets, toggle "Recommendations sidebar" off → return to YouTube; it's back — no session restart needed.
  4. Let it hit a break → all trimmed elements reappear.
ExpectTrims apply only during work/always-on; restored on break/idle; toggles update live.

F · Whole-site blocking

Hard-block entire domains during focus → redirect to the in-extension block page. Defaults: tiktok.com, facebook.com.

  1. Start a session.
  2. Visit tiktok.com during work → land on the block page ("This one can wait." + "Break in mm:ss").
  3. Add your own domain under Hard-blocked sites (e.g. example.com → Add) → visit it during work → blocked.
  4. When the break starts, the block page auto-returns you.
ExpectBlocks only during work/always-on; the block page countdown ticks to the break.

G · YouTube channel lock

Allow only specific channels; block every other YouTube video/channel during focus.

  1. Dashboard → YouTube channel lock → switch on → add a channel (@LofiGirl or a /channel/UC… URL).
  2. Start a session.
  3. Open a video from a different channel during work → redirected to the block page.
  4. Open the allowed channel → plays normally.
ExpectHome/search stay reachable (feed hidden by preset); only channel/watch pages are gated. Lock only bites when the list is non-empty.

H · Always-on focus

Blocking stays active regardless of any session.

  • Popup → toggle Always-on focus on (no session needed) → visit tiktok.com → blocked; the block page shows ∞ instead of a countdown. Toggle off → unblocked.

I · Exit friction no easy off-switch

Ending a session early is deliberately hard.

  • In Work or Break, click End session (or End in the popup).
ExpectA modal appears — confirm stays disabled for an 8s countdown and until you type i am giving up. "Keep going" cancels. Only then does it stop.

J · Gamification & stats

Day streak, focus-this-week, sessions today, distractions blocked, XP/levels, 28-day focus wall, end-of-session roll-up.

Easy to verify now

  • Distractions blocked — each block-page hit increments (watch the dashboard card / Command tile update live).
  • Sessions today & Day streak — complete a session (let all blocks finish) → both increment; roll-up shows the summary.
!

XP, "Focus this week", and the focus wall need real minutes. They come from completed work-block minutes rounded to whole minutes. The Test rhythm's 6-second blocks round to 0, so they won't move on test runs — run a real block (e.g. 25/5, let one 25-min block finish) to see them change. Quirk of quick-testing, not a bug.

K · Themes

  • Dashboard top-right Dark/Light toggle. Applies across popup, dashboard, and block page; persists across reloads.

05

Cross-browser

Everything works the same in dist/chrome and dist/firefox (one codebase via the browser.* polyfill).

Worth a quick parallel run in Firefox of the block page (§F) and the YouTube trim (§E) to confirm parity.

06

Known caveats

So nothing surprises you during the pass.

  • Test rhythm + closed UI: very short (<~30s) transitions can lag if both popup and dashboard are closed (the browser clamps tiny alarms). Real-length sessions are unaffected.
  • XP / focus-wall need ≥1-minute blocks — test sessions won't move them (see §J).
  • Native apps & phone are out of scope — Lockd controls the browser only (stated honestly on the dashboard).
  • Other browsers / incognito — an extension can be disabled or bypassed by switching browsers; the friction is in-app, not OS-level.
  • Fonts load from Google Fonts; if blocked, the UI falls back to system fonts (layout holds).
  • Selectors are best-effort — YouTube/IG/X/Reddit change their DOM; a preset may need a selector update after a redesign (tracked in the v1.1 backlog).

07

Reset state

Clean slate when you want one. Stats, settings, presets, channels, and the current session all live in browser.storage.local.

  • Quickest: remove the extension and load it again.
  • DevTools: on the dashboard page → Application → Storage → clear → reload.

08

Fastest full pass

Covers all 8 phases in about 3 minutes.

  1. Quick-start from the popup.
  2. Confirm YouTube trims + TikTok block page.
  3. Toggle a preset off and watch it restore live.
  4. Trigger the exit-friction modal.
  5. Enable always-on and hit a block page (∞).
  6. Run Test / Test with the dashboard open → watch the full loop land on the roll-up.
